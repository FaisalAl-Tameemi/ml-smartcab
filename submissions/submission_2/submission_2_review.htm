Identify and update state

Student has identified states that model the driving agent and environment, along with a sound justification.

Well done identifying and justifying which input variables you included in the state space: "Namely, next_waypoint for directions, light for traffic light signal and oncoming traffic." These states are necessary in order for the agent to gain enough critical information about the environment to make decisions and thus eventually learning the optimal rules of the road.

Furthermore, to your point, the only way to really include deadline is to make this a boolean variable; given the large number of states that the deadline variable can take thus significantly increasing the complexity of our state space, the agent will not be able to feasibly explore all of the states to learn the optimal action within in 100 trials.

You mentioned the following "Note: This implementation chooses to ignore right and left because we can safely assume that other agents (cars) on the grid will follow the driving rules outlined." -- What if the agent learned the wrong policy and haphazardly ran the red light and made a left turn? Ideally, this scenario should not occur, but on the other hand, it does not mean that it won't. Also, your completely right about omitting input['right'], given the right-a-way traffic rules; I would just rethink traffic from the left here in addition to further explain your justification.

The driving agent updates its state when running, based on current input. The exact state does not matter, and need not be correlated with inputs, but it should change during a run.

The states correctly update:

# Update the current state
self.state = (

Implement Q-Learning

The driving agent updates a table/mapping of Q-values correctly, implementing the Q-Learning algorithm.

Q-learning is correctly implemented in your code. Nice!

 Q_hat = self.QTable[previous['state']][previous['action']]
Q_hat = Q_hat + (self.alpha * (previous['reward'] + (self.gamma * (max(self.QTable[state].values()))) - Q_hat))

Given the current set of Q-values for a state, it picks the best available action.

The best available action is selected based on the maximum Q-value for the corresponding states. Good idea to add a tie-breaker if more than one maximum Q-value is present by randomly selecting the action to take.

max_reward_actions = [ action for action, Q_hat in self.QTable[state].items() \
       if Q_hat == max(self.QTable[state].values()) ]
# choose randomly if more than 1 argmax action
return random.choice(max_reward_actions)

Additionally, nicely done implementing the epsilon-greedy policy to address the trade-off between exploiting the information stored in the Q-table and exploring the environment by occasionally taking a random action in order to learn the value of that action for the corresponding state

Excellent analysis provided for each experiment performed in which different tuning parameters where tested. Did you look into decaying epsilon over time? I ask this because in your code you include the following:

# TODO: make epsilon a decaying value as there is no need for it
#in the last 10 trials
self.epsilon = epsilon # exploration rate

Epsilon does not decay, rather remains static at 0.85. It would be a good idea to decay epsilon as it works relatively well here. Basically, you would reduce the frequency of entering the "exploration" phase over time. Therefore, you could look into implementing a decay factor for epsilon over time as eventually the agent needs to stop exploring and strictly exploit the information learned in the Q-table; especially in the latter half of the simulation.
