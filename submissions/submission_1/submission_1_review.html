Identify and update state

Student has identified states that model the driving agent and environment, along with a sound justification.

Although this admittedly isn't visualized in the simulation window, the streets are actually two-way streets, so the justification for ignoring either inputs['left'] or inputs['right'] is not valid.

Note: regardless of that, inputs['right'] can be excluded from the state safely. This can be easily seen after examining the relevant traffic rules. (Obeying the traffic light prevents any possible collision with the car on the right.)

It also seems that the first part of this answer is not consistent with the code, as this is how you describe the state that is used:

This implementation of the smart agent will build a state from a subset of that the information above. Namely, next_waypoint for directions, light for traffic light signal and right (or left) for sideway traffic, and deadline.

While in the code, the following variables are not used:

  • deadline
  • inputs['left'] or inputs['right']

On the other hand, inputs['oncoming'] is used.

The driving agent updates its state when running, based on current input. The exact state does not matter, and need not be correlated with inputs, but it should change during a run.

Implement Q-Learning

The driving agent updates a table/mapping of Q-values correctly, implementing the Q-Learning algorithm.

Nice work, the code is very easy to read thanks to the separate QLearner class.

Given the current set of Q-values for a state, it picks the best available action.

Note: usually, epsilon is the probability of taking a random action, while in your implementation it is the reverse. This can be a bit confusing on first glance.

Suggestion: consider using a decaying epsilon value; it doesn't really make sense to take random actions during the last 10 (evaluation) trials, as by that time your agent has enough knowledge to perform well anyway.

Student has reported the changes in behavior observed, and provided a reasonable explanation for them.

It's a very good idea to discuss the implementation details here like you did, but you also have to answer the first question: "What changes do you notice in the agent's behavior when compared to the basic driving agent when random actions were always taken?"
For example, you should mention that:

  • The Q-Learning agent starts to follow the rules of the road after some trials, while the random agent keeps ignoring them
  • The Q-Learning agent starts to follow the recommendations of the planner, while the random agent keeps wandering aimlessly

Enhance the driving agent

The driving agent is able to consistently reach the destination within allotted time, with net reward remaining positive.

Specific improvements made by the student beyond the basic Q-Learning implementation have been reported, including at least one parameter that was tuned along with the values tested. The corresponding results for each value are also reported.

You've thoroughly described the tuning process, and also provided some insights about the reasons behind the outcomes (i.e. the deterministic nature of the rewards/penalties).

Suggestion: because your agent is optimistic about unexplored (state, action) pairs (due to the positive initial Q-value), having an epsilon value that isn't 1 is superfluous, and it also forces the agent to take random actions occasionally, even when it has explored a particular state completely.
Although the difference between a 0% chance and 15% chance ofâ€‹ taking a random action might not be noticeable using the success rate as the sole evaluation metric (as a single suboptimal action doesn't compromise the success of a whole trial), I'm sure that other metrics could indicate a significant improvement.

A description is provided of what an ideal or optimal policy would be. The performance of the final driving agent is discussed and compared to how close it is to learning the stated optimal policy.

The questions of this section are not really answered in the Project Report: "Does your agent get close to finding an optimal policy, i.e. reach the destination in the minimum possible time, and not incur any penalties? How would you describe an optimal policy for this problem?"

The second question is fairly straightforward to address, by describing the optimal policy in your own words.

The first one is a bit trickier; your answer must compare the final learned policy of the agent with the optimal policy along both of the main goals of the agent:

  • Does your agent always obey the traffic rules during the last 10 iterations? (Hint: the agent rarely encounters other cars at an intersection, so it has trouble learning how to act when it does happen.)
  • Does the agent always follow the suggestions of the planner? (Hint: a high gamma value could make the agent go in circles, as that results in it accumulating rewards after the initial -0.5 penalty.)
